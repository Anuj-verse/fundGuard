# FundGuard Microservices Documentation

## 1. Edge Service (`services/edge-service`)
**Role**: High-speed, real-time transaction ingestion and machine learning inference.

### Files & Components:
*   `app/main.py`
    *   **Description**: FastAPI entrypoint and request router. Initiates the ONNX scorer and feature extractor on startup.
    *   **Key Imports**: `FastAPI`, `confluent_kafka`, `prometheus_client`, `app.kafka.KafkaPublisher`
    *   **Interface**: `POST /score`, `GET /health`, `GET /metrics`
*   `app/scorer.py`
    *   **Description**: Loads the exported XGBoost ONNX model and executes sub-10ms risk inference.
    *   **Key Imports**: `onnxruntime`, `numpy`
    *   **Interface**: `OnnxScorer`, `OnnxScorer.score(features)`
*   `app/features.py`
    *   **Description**: Computes 18 core model features from the incoming transaction payload (e.g., amount ratios, time-delays).
    *   **Key Imports**: `pandas`, `app.schemas.TransactionRequest`
    *   **Interface**: `FeatureExtractor`, `FeatureExtractor.extract(request)`
*   `app/kafka.py`
    *   **Description**: Asynchronous producer streaming evaluated transactions into the Kafka bus.
    *   **Key Imports**: `confluent_kafka.Producer`, `json`, `logging`
    *   **Interface**: `KafkaPublisher`, `KafkaPublisher.publish(payload)`
*   `app/schemas.py`
    *   **Description**: Pydantic schema validation for typed request/response payload parsing.
    *   **Key Imports**: `pydantic.BaseModel`
    *   **Interface**: `TransactionRequest`, `TransactionLiveEvent`
*   `app/metrics.py`
    *   **Description**: Instrumentation hooks exposing Prometheus-compatible latency distributions and counters.
    *   **Key Imports**: `prometheus_client`
    *   **Interface**: `REQUEST_COUNT`, `LATENCY_HISTOGRAM`

## 2. Graph Service (`services/graph-service`)
**Role**: Complex network intelligence and fraud ring identification.

### Files & Components:
*   `app/main.py`
    *   **Description**: FastAPI management shell. Bootstraps the Neo4j connection and the Kafka background listener thread.
    *   **Key Imports**: `FastAPI`, `app.graph.Neo4jGraph`, `app.consumer.GraphKafkaConsumer`
    *   **Interface**: `GET /graph/{account_id}`, `GET /patterns/active`, `GET /health`
*   `app/graph.py`
    *   **Description**: Cypher query execution wrapper. Handles graph data merges for sender/receiver accounts, devices, and transactional relationships.
    *   **Key Imports**: `neo4j.GraphDatabase`
    *   **Interface**: `Neo4jGraph`, `Neo4jGraph.ingest_transaction(txn)`, `Neo4jGraph.check_patterns(txn)`
*   `app/consumer.py`
    *   **Description**: Consumes transactions from `transactions-live`. Evaluates the payloads against graph traversal patterns and forwards findings to `graph-events`.
    *   **Key Imports**: `confluent_kafka.Consumer`, `json`
    *   **Interface**: `GraphKafkaConsumer`, `GraphKafkaConsumer.start()`, `GraphKafkaConsumer.stop()`
*   `app/gds.py`
    *   **Description**: Scheduler for Neo4j Graph Data Science algorithms (like PageRank and Louvain) to establish holistic scoring baselines.
    *   **Key Imports**: `neo4j`, `APScheduler`
    *   **Interface**: `start_scheduler()`

---

## 3. Risk Engine (`services/risk-engine`)
**Role**: Ultimate arbiter. Weighs inference probabilities against rigid operational logic.

### Files & Components:
*   `app/main.py`
    *   **Description**: Unified Risk Engine orchestrator. Fuses the sub-scores generated by the Edge and Graph services.
    *   **Key Imports**: `FastAPI`, `kafka.KafkaConsumer`, `kafka.KafkaProducer`
    *   **Interface**: Background thread consuming `graph-events` and producing to `risk-scores`.
*   `app/schemas.py`
    *   **Description**: Strict schema validation to ensure the payloads sent by ML and Graph teams match the operational definitions.
    *   **Key Imports**: `pydantic.BaseModel`
    *   **Interface**: `RiskScoreEvent`

---

## 4. Dashboard API (`services/dashboard-api`)
**Role**: System gateway aggregating Kafka events and exposing them to Web UIs via HTTP and WebSockets.

### Files & Components:
*   `app/main.py`
    *   **Description**: Exposes asynchronous WebSockets to push anomaly scores into frontends instantly, natively integrating cross-origin constraints (CORS).
    *   **Key Imports**: `FastAPI`, `WebSocket`, `aiokafka.AIOKafkaConsumer`
    *   **Interface**: `WS /ws`, `POST /api/explain`

## 6. LLM Service (`services/llm-service`)
**Role**: Generative AI system to convert rigid network and inference logs into human-readable Suspicious Activity Reports (SAR).

### Files & Components:
*   `app/main.py`
    *   **Description**: Entrypoint for NLP inferences parsing complex Graph/Edge rule-triggers.
    *   **Key Imports**: `FastAPI`, localized LLM inference engines (like Ollama or Transformers).
    *   **Interface**: Endpoint receiving anomaly records and returning detailed string explanations.
## đź§Ş Testing Services Individually

If you need to isolate and verify the behavior of a single service without triggering the entire end-to-end event pipeline, use these methods:

### 1. Edge Service
- **Unit/Integration Tests**: Run the localized tests to verify the ONNX models load and process accurately.
  ```bash
  cd services/edge-service
  pytest tests/
  ```
- **Manual Inference**: Shoot a mock REST request directly at the FastAPI container bypassing Kafka.
  ```bash
  curl -X POST http://localhost:8001/score \
       -H "Content-Type: application/json" \
       -d '{"transaction_id": "test_1", "sender_account_id": "A_1", "receiver_account_id": "A_2", "amount": 500, "currency": "INR", "channel": "WEB"}'
  ```

### 2. Graph Service
- **Cypher Validation**: Validate Neo4j queries directly using the FastAPI endpoints without sending transactions.
  ```bash
  # Check ego network
  curl http://localhost:8002/graph/ACC_NORMAL_01
  
  # Check cached patterns
  curl http://localhost:8002/patterns/active
  ```
- **Pipeline Bypass**: Directly push a JSON transaction object into `transactions-live` via `kafka-console-producer` inside the broker container to isolate ingestion issues.

### 3. Risk Engine
- **Health Verification**: Check that background consumer threads are alive.
  ```bash
  curl http://localhost:8004/health
  ```
- **Simulate Graph Events**: Inject a mock payload directly into the `graph-events` topic using the Kafka CLI, then tail `docker logs services-risk-engine-1` to watch it compile the unified score independently.

### 4. Dashboard API
- **WebSocket Hook**: Use a quick Python script or `wscat` to attach precisely to the API and print broadcasted risk events.
  ```python
  import asyncio, websockets
  async def listen():
      async with websockets.connect("ws://localhost:8005/ws") as ws:
          print("Listening...")
          print(await ws.recv())
  asyncio.run(listen())
  ```

### 5. LLM Service
- **Explanation Test**: Trigger the generative translation layer manually to verify LLM latency.
  ```bash
  curl -X POST http://localhost:8003/explain \
       -H "Content-Type: application/json" \
       -d '{"transaction_id": "txn_test"}'
  ```
